URL specification update area

I am working on a revision of RFC 1738 (Uniform Resource Locators) and RFC 1808 (Relative Uniform Resource Locators). This is my active work area, so don't be surprised if the documents change rapidly.

The work is progressing as follows:

  1. I grabbed the current set of URL RFCs and removed the header/footer garbage that makes diff comparisons difficult.
  2. I created a base document (a copy of the original content of RFC 1738 and RFC 1808) which merges the two specs without changing any of the effective content, except in that all of the scheme-specific definitions are removed. I was going to use it for diffs, but the extent of editorial changes necessary to really merge the two specs makes that impossible.
  3. The first complete draft is available.
  4. The second complete draft is available. I also made a diff from 00 to 01.
  5. I have created three test pages for relative URL parsing, parts one, two, and three, and have tested a few common browsers. I need more testers. If all (or almost all) of the browsers do things differently than the spec, and it isn't an obvious bug in those browsers, then the spec will need to change. Actually, it already has for draft 01 and 02.
  6. The third complete draft is available. This is my final draft -- it is in Larry Masinter's care from now on. I also made a diff from 01 to 02. Changes from the RFCs are noted in the last appendix.
